## Bindings

When using Pluot across programming language boundaries, we need to consider bindings in both directions (to/from).

### Bindings _to_ (the Rust program)

The non-Rust environment *calls* these.

#### `render`

Or its bound variant (e.g., `render_py`, `render_wasm`, etc.) which handles serialization/deserialization or other platform-specific things.

#### `pick`

Perform "picking" to look-up the value at a specific (relative) screen coordinate.
Used during hover and click event handling (e.g., to show tooltips).

### Bindings _from_ (the non-Rust program)

The non-Rust environment *provides* these (for Rust to call).

#### `log`

Call the platform-specific logging function (e.g., `console.log` on WASM, `logger.info` on Python, etc.).

#### `get_font`

WASM-specific.

Get a font from the system environment. (This avoids the need to bundle a bunch of fonts, which would both impact the bundle size and run into font licensing challenges).
